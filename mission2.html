Mission 2 : Interface et Indicateurs

L’objectif de cette mission est de commencer à exploiter la base de données de l’entreprise KDou pour pouvoir par la suite avoir des analyses et indicateurs pertinents. On souhaite faire des requêtes avec le logiciel Low Code Access. Ces requêtes va nous permettre d'analyser les donnes de KDou plus correctement.

Indicateurs analysés

Indicateur 1 : Le prix moyen de vente des produits, le prix minimum et le prix maximum.

Énoncé : Analyse de prix des produits que KDou vend

Interprétation : Il faut comparer les prix des produits.

Résultat :

Code SQL :
SELECT AVG(PrixUnit) AS PrixMoyen, MIN(PrixUnit) AS PrixMinimum, MAX(PrixUnit) AS PrixMaximum
FROM Produit;

Indicateur 2 : Le chiffre d’affaire total généré avec les informations dont on dispose.

Énoncé : Calcul de chiffre d'affaires.

Interprétation : Le chiffres d'affaires est important pour les analys.

Résultat :

Code SQL :
SELECT NomProd
FROM Produit
WHERE QteParUnite LIKE '*carton*' OR QteParUnite LIKE '*bouteille*';

Indicateur 3 : Des clients qui n’ont pas encore passé de commande

Énoncé : Analyse d'état des clients

Interprétation : Pour connaitre mieux nos clients, faut voir qui sont actif et commandent.

Résultat :

Code SQL :
SELECT SUM(DetailCommande.PrixUnitCli * DetailCommande.QteCom * (1 - DetailCommande.Remise)) AS ChiffreAffaireTotal
FROM DetailCommande;

Indicateur 4 : Les produits conditionnés dans du carton ou dans du verre

Énoncé : Pour être "éco-friendly", faut regarder ce que kdou vend.

Interprétation : C'est important de savoir le quantite du carton ou verre que KDou vend pour leur objectif d'être "éco-friendly"

Résultat :

Code SQL :
SELECT Client.ContactCli
FROM Client
LEFT JOIN Commande ON Client.CodeCli = Commande.CodeCli
WHERE Commande.NoCom IS NULL;
